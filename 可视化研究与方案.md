# 可视化与交互方案（研究支撑版）

> 来源：4 个并行研究 agent（叙事工具范式 / 知识态表达 / InfoVis 选型 / 未来能力技术与UX），均带一手出处。
> 一句话：**力导向"节点+连线"图不是最佳表达；研究一致指向「认知矩阵为主视图 + 多镜头联动」。** 更深一层：数据模型应顺势升级成未来能力（对抗agent/生成/创作检查/LLM自检）的地基。

> **⚠️ 本文仍然有效，但 §1 的结论已据实测修订（2026-06）。** 这份研究的洞见（矩阵 > 节点图、CMV/brushing、外部锚定、可能世界云）依然是设计依据；§2/§3 是后续 roadmap。
> 唯一被改的是「谁当 hero」：把认知矩阵当**唯一主视图**经用户体验反馈否决（太抽象、不适合玩家第一眼、列无标签需逐个点）。最终架构 = **「一引擎两面」**——**关系网当玩家(现场)模式的直观入口**、**认知矩阵归创作者(上帝)模式当分析/查逻辑工具**，底部一根幕轴统一时间。即：矩阵没被丢弃，而是**搬到了它真正强的地方**。现状见 `引擎与Demo设计.md` §0.5 与 `README.md`。

---

## 0. 研究铁证（为什么换掉图）

- **Ghoniem / Fekete / Castagliola（IEEE InfoVis 2004 + Information Visualization 2005）对照实验**：图节点 **>~20 时，邻接矩阵在几乎所有任务上优于 node-link**，唯一例外是"找路径/追踪链条"。Okoe/Kobourov(TVCG 2018) 大规模复现；二部数据(arXiv 2208.04458)结论：matrix 最稳健。
- 我们的数据 = 角色 × ~200 事实、**稠密、属性化、实体>20** —— 这是 node-link 的教科书级最差情形、矩阵的最佳情形。各种"剪边/捆绑/聚合"对本就稠密的情形只是创可贴，文献直接指向**换编码**。
- 业界佐证：Obsidian 全局图被公认是"好看但无用的 hairball"；Twine 地图一旦有变量就"浑浊漏路径"。

## 1. 推荐方案：一套"随任务切换的联动镜头"（CMV + brushing/linking）

不要一张全能图。每个任务用它最优的表达，全部挂同一份模型、同一套选择总线、同一套三态编码。

| 任务（你想回答） | 最佳表达 | React 库首选 | 依据 |
|---|---|---|---|
| **谁知道什么 / 隔离墙 / 假信念 / 反转** ⭐主视图 | **认知矩阵**（行=角色，列=事实，格=四态） | 自绘 SVG/Canvas + d3-scale（要重排）/ 或 `@nivo/heatmap`(canvas) | Ghoniem：稠密二部矩阵最优 |
| 人物关系 / 找路径 | **关系网 node-link**（只人物+关系，稀疏） | `@xyflow/react`(React Flow) + ego聚焦 | node-link 唯一稳赢的任务=路径 |
| 主线随幕推进 | **时间线 / 幕泳道**（幕为列，真相/线索/演绎三轨） | `visx` 自绘泳道（离散分幕）/ 或 vis-timeline | Munzner：时间=位置编码 |
| 线索如何推出真相（可解性，创作向） | **分层 DAG**（线索→事实→阴谋） | React Flow + `@dagrejs/dagre` | 因果/层级=分层布局 |
| 某角色全部信息 / 审问 | 档案面板 + 对话（已有） | — | — |

### 三态/四态编码（全镜头统一，fog-of-war 借 RTS 三层）
| 状态 | 编码 |
|---|---|
| 当前已知（真相） | 全彩、实心、`✓` |
| **假信念（误信）** | 警示色(红) + `✗` + 矛盾徽章；真值藏在背后幽灵层 ← 戏剧高潮 |
| 不知 / 墙后 | 暗/空 = 隔离墙（SVG mask 黑色区） |
| 过时（曾知、现已为假） | 去饱和灰雾 |
颜色 + 格内字形**双编码**（色盲友好）。

### 认知矩阵的"惊艳"要点（这是 demo 新主角）
- 行列**重排序(seriation / Bertin 可重排矩阵)**：让"知识阵营、信息孤岛"自动浮成色块，**误信落在对角线之外**——一屏看尽 fog-of-war。
- 列按"第几幕揭示"排 → 拖幕滑块，矩阵**从左到右一格格亮起**（时序也有了）。
- 一键**「只看假信念」** = 剧本杀的高潮视图。底部固定一行**「上帝视角」= 真相全集**，对照隔离墙。
- 点格 → 详情 + 其它镜头联动高亮。

### 联动架构（brushing & linking）
- **单一选择总线**：Zustand store 存 `{selectedChars, selectedFacts, selectedAct, hovered}`；每个镜头既消费(高亮)又生产(点击/刷选)。
- **共享 ID 空间**：人物/事实/线索/幕都用稳定 ID（我们已有）。
- 典型流：关系图点角色 → 矩阵高亮该行 → 时间线高亮其已知事实的解锁幕 → DAG 高亮指向这些事实的线索链。
- 遵守 Baldonado CMV 准则：只在互补时加视图；未选=去饱和而非隐藏（保上下文）。

## 2. 更深一层：把数据模型升级成"未来能力的地基"（最关键）

四个未来需求（①对抗 agent ②生成变体 ③创作检查 ④LLM自检）反复指向同一组数据升级。**贯穿性元结论：凡"让 LLM 判自己"都不可靠（自我纠错常越改越差，Huang et al. ICLR 2024）；可靠性来自外部锚定——真正的 SAT/ASP 求解器、把真相图谱当参考答案、异构评审团。**

| 升级 | 怎么做 | 服务于 |
|---|---|---|
| **信念 schema**（借 Talk of the Town / SymbolicToM） | 每条信念带 `owner/subject/value/evidence(来源+类型)/strength/accuracy(=假信念标记)/predecessor(曾信什么)` + **`from_act`(第几幕获知)**；嵌套信念=每层一张图(封顶深度2) | 矩阵/对抗/检查 |
| ↳ 直接收益：**认知卡随幕生长** | 有了 `from_act`，现场模式「认知」tab 才能按当前幕只显示 TA 已获知的信念（背景=开局即知、局中新知逐幕点亮），与「本幕/线索/矩阵幕前沿」一致。*（现状：认知 tab 是全集、不随幕变——源数据缺 `from_act`）* | ①迷雾时间维一致性 |
| **真相图谱→可被求解器读的约束** | 每条"线索→事实"声明蕴含/排除哪些事实 → 可导出 ASP/SAT；建"可解性求解服务"：是否唯一可解 + 每条线索是否必要 | ③可解性、②上线闸门 |
| **知识分区=信息流控制(IFC)一等公民** | 每个(事实,角色)打可见性标签；运行时 agent 上下文**只注入它该知道的**——防泄漏靠"上下文里根本没有"，不靠 prompt 叮嘱（Sotopia-ToM：9.9%→2.2%） | ①防泄漏、③演绎公平性验证 |
| **显式时间/因果轴** | 分幕硬化成时间线 + 事件间因果边(启用/矛盾) | ③plot-hole 检测（变成图上查环，比读长文可靠） |
| **事件溯源 + 可分支** | 世界状态=不可变事件日志；"另一条故事线"=某事件后分支 | ②变体生成 + UI 回滚（Loom 式多元宇宙树） |
| **生成被约束+检索锚定** | 计划层(图谱/大纲)约束 LLM 生成；GraphRAG 过滤"引用不存在实体"的输出 | ①②不跑偏/不幻觉 |
| **每节点带 provenance** | `source(人/AI生成/AI改)/置信度/通过了哪些检查/时间戳` | ④审查、diff、内容凭证 |

**校验流水线**（④）= 求解器(可解性) + NLI/QA事实核对(SummaC/QAGS/FActScore) + 异构评审团(PoLL，比单GPT-4更准更便宜) + 采样发散查幻觉(SelfCheckGPT)，**真相图谱当锚点**；显式避免"同一 LLM 写完自己改"。

## 3. 可选的"封神级"惊艳视图（后续阶段）

- **每角色「可能世界云/透镜」（动态认识逻辑 DEL/Kripke）**：每个角色背后是一团"仍可能为真的世界"；揭示线索 = **动画剪枝**掉不兼容的世界；**假信念 = 整团云坍缩到一个错误的世界**（如黛利拉所有箭头都指向"阿奇还活着"）。最电影感、最能把"反转"演出来。
- **谎言/流言传播图（Talk of the Town）**：一条事实/谎言沿角色→角色传播，按证据类型上色（观察=实线/陈述=细线/谎言=红虚线）；切换"他们相信的 vs 真实轨迹"戏剧性揭穿谁撒谎。
- **微交互**：SVG mask 落雾 + 聚光揭示；CSS 3D 卡片翻转做单条"表象→真相"；GSAP Flip 做"反转重连"的节点滑行。
- **反泄漏机制借 Obra Dinn**：阈值批量确认（N条同时对才盖章，绝不单格验证，防穷举）；用排版/材质而非颜色泄漏对错。
- **Outer Wilds Ship Log 的两条铁律**：①颜色=故事线(不是状态)，让人"盯一种颜色追一条线"；②永不一次显示全图、节点手工布局、只随发现增长。

## 4. 建议落地优先级

**第一阶段（现在·UX 主体，直接回答"怎么呈现最好"）**
1. **认知矩阵（主视图/hero）**：四态编码 + 重排 + 幕滑块逐列点亮 + 「只看假信念」+ 上帝视角对照行。
2. **关系网**（React Flow，只人物+关系，ego 聚焦 + 联动）。
3. **时间线/幕泳道**（visx 自绘）。
4. 三镜头 + 档案 + 对话，全部挂 **Zustand 选择总线**，统一三态编码与 brushing。

**第二阶段（地基·随创作/对抗路线）**
5. 信念 schema + 知识分区→IFC + 时间/因果轴（先把"可解性求解服务"和"一致性检查"做出来，给③创作工具确定性价值）。
6. 推理链 DAG 视图 + 可解性/一致性/公平性校验流水线（外部锚定）。

**第三阶段（封神/对抗）**
7. 可能世界云透镜 + 谎言传播图；事件溯源可分支 + 多元宇宙树 UI；对抗多 agent（导演/裁判 + IFC 上下文 + 定期重注入人设防漂移）。
