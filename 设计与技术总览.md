# 《流氓叙事》设计与技术总览

> **单一事实源**。把一个真实剧本杀（《流氓叙事》，2281 页纯图本）数字化成「时序、多视角的知识图谱」，在其上长出两类产品。本文是 **设计 + 关键技术 + 现状** 的唯一总览；**运行/demo 走法见 `README.md`**。
> 护城河不是 AI，而是这套**结构化、可计算**的模型：引擎在每一幕都算得清「每个角色知道什么、被瞒什么、手上有哪些线索、要守哪些秘密」，以及「改一处会牵动什么、改完还自不自洽」。

---

## 1. 一引擎两面 · 核心抽象 `G(幕, 视角)`

**一台引擎**：剧本 → 知识图谱 → 一个对象 **`G(act, perspective) → 子图`**。
- **Z 维（幕/时间）**：每条 事实/线索/关系/事件 带「第 N 幕起可知」，图沿 Z 生长。
- **XY 维（某幕切片）**：那一刻谁知道什么、关系如何、桌上有哪些线索。
- **视角维**：`G(act,上帝)` = 截至该幕全部真相；`G(act,角色)` = 该角色「知道/相信」的子图（含假信念）。**隔离墙 = `G(act,上帝) \ G(act,角色)`**（例：「她杀的延迟=阿奇」在任何 `G(act,黛利拉)` 中都不存在）。

**两个面 = 同一份图的两个投影，不是岔路**：
- **现场·入局（玩家·迷雾）**：关系网当主舞台 + 认知卡 + 防泄漏对抗 agent。
- **上帝·创作（作者·全知）= 创作画布**：可追因果的故事图，读懂 / 改本 / 查逻辑 / 看涟漪。

## 2. 它解决什么业务痛点（痛点 → 能力 → 技术 → 状态）

> 设计立场（与下文「分工铁律」一致）：**骨架（世界观/人物关系/关键节点）由人定，AI 在「定稿会之后」的细纲/写作阶段批量补文本细节、纠逻辑、查一致，作者专注「上皮」润色与品控**——AI 提议、人拍板、求解器盖章。下表「状态」诚实区分已建成与待做。

| 业务痛点 | 我们的能力 | 关键技术 | 状态 |
|---|---|---|---|
| **剧本创作慢**：成熟作者 1 本/月；全公司仅 2 位强「上皮」作者；新人转型需 2–3 月培训 | 创作画布把本子结构化成可追因果的故事图；体检台自动查逻辑/一致；真正改本子（AI 提议 + 连锁改写、作者只品控）。目标：成熟作者 1→2–3 本/月、培训周期减半、月立项 3→5–6 本 | 故事图投影；确定性图检查 + clingo ASP 求解器；确定性依赖传播 + LLM 提议 + 求解器盖章 | 已建成 |
| **板子迭代贵**：每店 2–3 名板子作者，3–4 天~1 周换一版，上海店预计 5+ 人 | 锁死主线下批量生成演绎变体，「1 作者 + 1 电脑」管 10 个角色的全部板子，作者只做品控 | 改本/连锁改写 + 求解器盖章（锁主线 = 不破可解性）；防泄漏 agent 演绎 | 地基已有，批量变体生成待做 |

## 3. Killer feature（技术 + 实例）

**从图本到可计算图谱**　将 2281 页纯图本转录、结构化为「实体 / 命题（事实，F-id）/ 线索 / 每角色知识分区（已知·假信念·须隐瞒）」；事件因果层从裁判文档抽取、逐条回溯原文核验后入图。每个节点带稳定 ID + 幕/视角元数据，故任意视图 = 一次投影查询（某角色主线 = `actor==X OR perceiver==X`，按幕排序），而非人工排版——这是后续所有呈现与推演的底座。规模：324 节点 / 125 条知识边 / 92 条因果边。

**结构性防泄漏对话**　不直接把剧本交给模型：先按**信息流隔离（IFC）**为每个角色编译专属知识库，剥离其不应知的全部裁判真相，再据此生成上下文——模型在结构上无法说出不在上下文里的真相；每轮回答附「依据（取自哪条主观认知）/ 是否触墙」，并以泄漏审计（关键真相 token 命中 = 0）兜底。假信念按"她信以为真"注入、真值剥离。

> 例：问黛利拉「阿奇是不是你杀的」——其上下文不含「阿奇已死」，只能依假信念「阿奇在出远海」回避；审计判定触墙、无泄漏。

**确定性依赖传播（划范围）**　改一处 → 沿因果边 **BFS 分层**求受影响集，基于**依赖追踪重算（truth maintenance / TMS）**衍生；是 over-approximation（用于划范围/高亮，不判生死）。迷雾即剪枝：改动在某角色因果可达之外，其认知不变。呈现为「改动涟漪」逐层激活点亮动画（激活扩散）+ blast-radius 度量（波及事件/牵动角色/幕跨度/是否触及结局）。纯确定性、无模型参与、可解释。

**受约束的剧情生成（生成 + 求解）**　改本/续写不让模型自由发挥，而是三段：模型仅**提议**内容 → 确定性引擎**划出**受影响范围 → **约束求解器（ASP / clingo）**对改动后整图重验可解性与一致性，通过才采纳——生成自带逻辑护栏、不破主线；仅会话生效、不改原始数据。同类约束求解（SAT/ASP）广泛用于航班排班、芯片验证等。
> 例：将「电台·挑火」改写为「两族和解」，引擎定位 30 处下游；连锁提议 drop「电台争夺引发屠杀」及其孤儿、rewrite「金狮医院」；求解器重验整条因果链后盖章。

**自动逻辑体检**　把可解性/一致性编码成约束，由约束求解器 + 图算法静态体检，定位长文中难察的逻辑洞——结论由求解器给出、不让模型自评（模型查漏随内容变长而退化）。

> 例：自动报「1995 vs 2000 屠杀口径矛盾」（核对属实）与「某谜底不可达」（实为漏录一条因果边）；补边后求解器复验通过。

**可视化选型**　追因果链/影响传播用节点图（分层 DAG），稠密关系（谁知道什么）用矩阵——信息可视化的经典对照结论（Ghoniem & Fekete）；多视图共享一份模型与一套选择总线（brushing & linking）。

**信念时序（地基·规划中）**　给每条信念加来源 + 获知幕（`from_act`），使认知卡随幕生长，并支撑「改本 → 角色认知」的完整投影。

## 4. 技术栈 + 仓库结构（前后端分离 · pnpm monorepo）

```
frontend/   Vite + React 19 + React Flow + elkjs + 自绘 SVG + Tailwind + Zustand   纯 UI
backend/    Node + Hono                                                          图编译/查询 + 对话 + 故事图/体检/改本
shared/     Zod schema + sliceGraph(G(幕,视角)) + StoryGraph                      前后端单一事实源
digitized/  数据（已全本数字化）+ _ocr（OCR 文本）+ 08_story（事件因果层）
```

- **后端**：Node + Hono（轻、TS 原生）；`compile.ts`(知识图谱 324 节点) · `story.ts`(故事图 68 事件/92 边) · `chat.ts`(防泄漏 prompt + leak/grounding) · `audit.ts`+`solver.ts`(体检：图检查 + clingo) · `cascade.ts`/`preview.ts`/`suggest.ts`(改本：连锁/预览/编剧)。
- **前端**：React Flow + elkjs（因果路径任务 node-link 最优）；自绘 SVG（总览泳道/矩阵镜头）；Zustand 跨视图选择总线。
- **LLM**：商用对话模型，OpenAI 兼容端点直连（可替换）；key 在 `.env`、不入库。
- **求解器 = clingo-wasm**（clingo 5.8 纯 wasm，无系统二进制）。
- **零数据库**：`digitized/` → 内存图 → filter 出 `G(幕,视角)`。多本子/多人 session 时再上图库。

## 5. 现状 + Roadmap

**已建成（demo 可跑）**
- 现场面：关系网 + 认知卡（线索含 OCR 卡面）+ 防泄漏对抗 agent + 命门追问 + 依据/墙后 + **同台对质**（2–6 角色各持迷雾 KB 当众对撞，裁判洞察谁在瞒/装不知/说谎）。
- 创作面：故事图 L1 总览 + L2 钻取详图（因果邻域/推演下游/溯源上游 + 焦点锚定/导航栈/「你在这里」）。
- **P3 创作体检台**：确定性图检查 + clingo ASP 求解器，外部锚定。
- **P4b 真正改本子**：插入/编辑/删除 + AI 编剧提议 + AI 全量连锁改写下游（分批增量）+ 求解器盖章 + 应用层（仅会话）。右侧贴边面板、悬停联动、实时态。
- **角色层 Phase 1（两面初步闭环）**：
  - **A1 改本 → 角色认知·影响面**：改/删/插事件 → 确定性派生哪些角色的认知被牵动（在场/知道/误信/守秘 + 具体事实），分「直接牵动 / 下游可能波及」两档，画布悬停刷亮。复用下游 BFS + 知识图谱 KNOWS/误信/守秘边，零求解器。
  - **B1 加人物·命名充实**：起名(+方向) → LLM 在阴谋网里充实成主角量级的人（5–7 条稠密关系 + 4–6 个带类型、跨多幕、因果锚定到现有事件的剧情线）→ 引擎框定（关系连在册真人、事件落合法幕、锚点是真事件、禁用重名）→ 采纳后**落进关系网**（节点 + 关系线枢纽）**+ 故事图**（带类型事件接进现有因果，矩阵左列出现新角色泳道）。仅会话、不落盘。

**接下来**
- **A2 改本 → 认知·完整可玩**：重放事件做 product-update，重算每角色迷雾 KB → 用改过的本子直接入局，agent 认知当场变。
- **信念 schema 地基**（`B+/B-/U` + `from_act` + provenance）：认知卡随幕生长，是 A2 与「加人物可玩」的共同地基。
- **B2 加人物·全新推荐**：AI 分析阴谋网缺口 → 提议契合的全新候选角色（不靠作者命名）。
- **角色层求解器闭环**：把新角色 + 新剧情纳入后端改本连锁 / ASP 盖章（可知性/时序/关系无矛盾/IFC），目前新增内容只在前端会话层渲染。

> **统一性**：这套把「创作体检台 = 求解器层」「信念 schema = 节点 provenance」「认知矩阵 = 画布一个镜头」「对抗 agent = what-if 重仿真层」收敛进同一张图。

## 6. 理论依据与出处（按需查阅）

> 扎根于叙事  / 信息可视化 / 知识表示的**成熟理论**（部分为验证数十年的奠基经典），并对齐 **2024–2025 最新前沿**。以下「前沿在前、奠基在后」。

- **近两年前沿**：Huang et al. ICLR 2024（LLM 自我纠错越改越差）· FlawedFictions 2025（arXiv 2504.11900，LLM 查漏随长度退化、注入 >50% 新漏洞）· PoLL 2024（异构评审团）· Sotopia-ToM（IFC 防泄漏 9.9%→2.2%）
- **求解器/可解性**：clingo / Potassco（ASP）· ASP-for-PCG（Smith & Mateas）
- **表示模型**：Trabasso（因果网络）· Swartjes & Theune（fabula 6 类）· Elson（Story Intention Graph）· Ryan / Kreminski Felt（story sifting）
- **依赖传播/重规划**：Doyle、de Kleer（TMS / ATMS）· AGM（信念修正）· Riedl & Young（IPOCL）· Fox（plan-stability）
- **可视化**：Ghoniem & Fekete（矩阵 vs node-link）· Sugiyama / StoryFlow（分层布局）· Collins & Loftus（激活扩散）
- **库**：@xyflow/react · elkjs · clingo-wasm · Zustand
