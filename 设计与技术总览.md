# 《流氓叙事》设计与技术总览

> **单一事实源**。把一个真实剧本杀（《流氓叙事》，2281 页纯图本）数字化成「时序、多视角的知识图谱」，在其上长出两类产品。本文是 **设计 + 关键技术 + 现状** 的唯一总览；**运行/demo 走法见 `README.md`**。
> 护城河不是 AI，而是这套**结构化、可计算**的模型：引擎在每一幕都算得清「每个角色知道什么、被瞒什么、手上有哪些线索、要守哪些秘密」，以及「改一处会牵动什么、改完还自不自洽」。

---

## 1. 一引擎两面 · 核心抽象 `G(幕, 视角)`

**一台引擎**：剧本 → 知识图谱 → 一个对象 **`G(act, perspective) → 子图`**。
- **Z 维（幕/时间）**：每条 事实/线索/关系/事件 带「第 N 幕起可知」，图沿 Z 生长。
- **XY 维（某幕切片）**：那一刻谁知道什么、关系如何、桌上有哪些线索。
- **视角维**：`G(act,上帝)` = 截至该幕全部真相；`G(act,角色)` = 该角色「知道/相信」的子图（含假信念）。**隔离墙 = `G(act,上帝) \ G(act,角色)`**（例：「她杀的延迟=阿奇」在任何 `G(act,黛利拉)` 中都不存在）。

**两个面 = 同一份图的两个投影，不是岔路**：
- **现场·入局（玩家·迷雾）**：关系网当主舞台 + 认知卡 + 防泄漏对抗 agent。
- **上帝·创作（作者·全知）= 创作画布**：可追因果的故事图，读懂 / 改本 / 查逻辑 / 看涟漪。

## 2. 它解决什么业务痛点（痛点 → 能力 → 技术 → 状态）

> 设计立场（与下文「分工铁律」一致）：**骨架（世界观/人物关系/关键节点）由人定，AI 在「定稿会之后」的细纲/写作阶段批量补文本细节、纠逻辑、查一致，作者专注「上皮」润色与品控**——AI 提议、人拍板、求解器盖章。下表「状态」诚实区分已建成与待做。

| 业务痛点 | 我们的能力 | 关键技术 | 状态 |
|---|---|---|---|
| **剧本创作慢**：成熟作者 1 本/月；全公司仅 2 位强「上皮」作者；新人转型需 2–3 月培训 | 创作画布把本子结构化成可追因果的故事图；体检台自动查逻辑/一致；真正改本子（AI 提议 + 连锁改写、作者只品控）。目标：成熟作者 1→2–3 本/月、培训周期减半、月立项 3→5–6 本 | 故事图投影；确定性图检查 + clingo ASP 求解器；确定性依赖传播 + LLM 提议 + 求解器盖章 | 已建成 |
| **板子迭代贵**：每店 2–3 名板子作者，3–4 天~1 周换一版，上海店预计 5+ 人 | 锁死主线下批量生成演绎变体，「1 作者 + 1 电脑」管 10 个角色的全部板子，作者只做品控 | 改本/连锁改写 + 求解器盖章（锁主线 = 不破可解性）；防泄漏 agent 演绎 | 地基已有，批量变体生成待做 |
| **扩张瓶颈**：4 店→年底 19 店，每店约 10 人，动线靠老师傅经验 | 动线模板化 + AI 辅助拆解（公共幕→小黑屋 1v1→公共幕），新店设计周期压缩、不冷场，19 店共用一套中枢 | 跨本子 SOP 框架；`G(幕,视角)` 调度 | 规划中（需 SOP 层 / 第二个本子） |
| **玩法同质**：剧情固定预写、一次性消费、复玩率低 | 对抗式互动剧——每个角色 = 一条独立故事线的 AI agent，真人现场提问/抛动作即触发角色间对抗、动态生成新分支，千人千面、高复玩、可线上延伸（新玩法 + 新收入线） | 防泄漏对抗 agent（投影 + IFC）；`G(幕,视角)`；（远期）多 agent + 事件溯源分支 | 单 agent 防泄漏底座已建成，多 agent 对抗/动态分支待做 |
| **线下情绪断点**（远期）：玩家与 NPC 禁止私联，线下情绪消费无法延续 | NPC 企微「数字陪聊」智能体 + 节日寄信自动化，把线下情绪延伸到线上、开新收入线 | 角色 agent 产品化（复用防泄漏投影） | 远期 |

注：以上经营数字来自早期 brainstorm，为方向性目标、非承诺；技术状态以本仓代码为准。

## 3. 关键技术与概念（load-bearing）

**一份客观因果图 + per-character 投影**　存**一份客观事件因果网**，每个角色只是它的**投影**（focalization）——不复制事件、只在 `(事件,角色)` 上挂可见性/认知；某角色主线 = 一句 `actor==X OR perceiver==X` 查询（story sifting）。节点 ~10 类（Event/Reveal/Decision/Lie/Perception/Goal/Outcome…），因果边 4 类（物理/动机/心理/使能）+ 叙事边（reveals/contradicts/depends_on）。**落地 schema 见 `shared/schema.ts`。**

**确定性依赖传播（框范围）**　改一处 → 沿因果边 **BFS 分层**求 impact set，基于**依赖追踪重算**（truth maintenance / TMS）衍生。是 over-approximation（"可能影响"、用于划范围/高亮，不判生死）。**迷雾即剪枝**：改动在某角色因果可达之外 → 其认知不变。这是「推演下游」动画与「连锁改写划范围」的骨架——**纯确定性、零 LLM、可解释**。

**外部求解器判定（判生死）**　把「剧本可解/自洽」编码成约束、交给真正的求解器 **clingo**（ASP，本项目用 `clingo-wasm`）判定——**同类约束求解（SAT/ASP）广泛用于航班排班、芯片验证等**，不让 LLM 自评。硬约束 = 谜底有支撑链可达、时序无悖论、无因果环、结局有前因；改动后重跑：有解=仍可解，**UNSAT/矛盾原子=精确定位被破坏处**。一个引擎同覆盖 一致性 + 可解性 + 定位。

**分工规则：LLM 提议 / 引擎划范围 / 求解器盖章**　**绝不让 LLM 自评影响或当一致性裁判**——LLM 自检随故事变长显著退化、自我纠错常越改越差（Huang ICLR 2024；FlawedFictions 2025）。所以改本三段：① LLM 只**提议**内容（编剧助手/连锁改写）→ ② 确定性引擎**划范围**（BFS 下游）→ ③ clingo **盖章**（before→after 重验）。LLM 只为红/黄项生成人话解释，不参与判定。

**IFC / fog-of-war 防泄漏**　每个 `(事实,角色)` 打可见性标签；运行时 agent 上下文**只注入它该知道的**——防泄漏靠「上下文里根本没有」，不靠 prompt 叮嘱（Sotopia-ToM：泄漏 9.9%→2.2%）。`BELIEVES_FALSELY` 按「她信以为真」注入、真值剥离，是**故意的不一致**、不可当 bug。

**node-link ↔ matrix（各用在哪）**　信息可视化的经典对照结论（Ghoniem & Fekete）：稠密关系（>20 节点）用矩阵、唯独「追路径/因果链」node-link 胜。所以 **因果链/影响传播 = node-link 分层图**（创作画布），认知矩阵降为「谁是知识枢纽」的可选镜头。多镜头共享一份模型 + 一套选择总线（brushing & linking）。

**改动涟漪动画（激活扩散）**　从改动点 BFS 分层、沿因果边**逐层激活点亮**（激活扩散，知识图谱检索里的经典做法）；blast-radius 度量（波及事件/牵动角色/幕跨度/**是否触及结局谜底**）。布局 = 幕为时间层 + 角色泳道 + **分层有向图**（ELK layered）。

**应用层·会话改本 + 连锁改写**　改本**仅会话生效、绝不写本子**（刷新即还原，demo 安全）：草稿 `Draft{增/删/改事件与边}` → 「应用」并入 `applied` 层 → `effectiveStory = 原图 + applied` 整图反映。**连锁改写**：改/删/插一个事件 → 秒算全部受影响下游（确定性、无 LLM）→ **分批并行喂 LLM、一边跑一边增量出**（去掉硬上限）→ 逐节点 keep/rewrite/drop（含传递推理：删 A → 依赖它的 B 判孤儿）→ 接受进草稿 → 求解器对整条链重新盖章。

**信念 schema / provenance（未来地基）**　给每条信念加 `owner/subject/value/evidence(来源+类型)/strength/from_act(第几幕获知)`。直接收益：① 认知卡随幕生长（现状是全集、不随幕变，因源数据缺 `from_act`）；② 改本→角色认知（P4c）有据可投。这块是「完整版可玩 + 时序认知」的地基。

> 封神级后续视图（钩子，不展开）：每角色**可能世界云**（DEL/Kripke，揭示线索=剪枝世界，假信念=云坍缩到错世界）；**谎言传播图**（事实沿角色传播、按证据类型上色）。

## 4. 技术栈 + 仓库结构（前后端分离 · pnpm monorepo）

```
frontend/   Vite + React 19 + React Flow + elkjs + 自绘 SVG + Tailwind + Zustand   纯 UI
backend/    Node + Hono                                                          图编译/查询 + 对话 + 故事图/体检/改本
shared/     Zod schema + sliceGraph(G(幕,视角)) + StoryGraph                      前后端单一事实源
digitized/  数据（已全本数字化）+ _ocr（OCR 文本）+ 08_story（事件因果层）
```

- **后端**：Node + Hono（轻、TS 原生）；`compile.ts`(知识图谱 324 节点) · `story.ts`(故事图 68 事件/92 边) · `chat.ts`(防泄漏 prompt + leak/grounding) · `audit.ts`+`solver.ts`(体检：图检查 + clingo) · `cascade.ts`/`preview.ts`/`suggest.ts`(改本：连锁/预览/编剧)。
- **前端**：React Flow + elkjs（因果路径任务 node-link 最优）；自绘 SVG（总览泳道/矩阵镜头）；Zustand 跨视图选择总线。
- **LLM = DeepSeek**（OpenAI 兼容端点直连，未用 AI SDK）：`deepseek-chat` 默认，`.env` 持 key（不入库）。
- **求解器 = clingo-wasm**（clingo 5.8 纯 wasm，无系统二进制）。
- **零数据库**：`digitized/` → 内存图 → filter 出 `G(幕,视角)`。多本子/多人 session 时再上图库。

## 5. 现状 + Roadmap

**已建成（demo 可跑）**
- 现场面：关系网 + 认知卡（线索含 OCR 卡面）+ 防泄漏对抗 agent + 命门追问 + 依据/墙后。
- 创作面：故事图 L1 总览 + L2 钻取详图（因果邻域/推演下游/溯源上游 + 焦点锚定/导航栈/「你在这里」）。
- **P3 创作体检台**：确定性图检查 + clingo ASP 求解器，外部锚定。
- **P4b 真正改本子**：插入/编辑/删除 + AI 编剧提议 + AI 全量连锁改写下游（分批增量）+ 求解器盖章 + 应用层（仅会话）。右侧贴边面板、悬停联动、实时态。

**接下来**
- **P4c 改本 → 角色认知（接回可玩）** ← 推荐：改事件 → 谁多/少知道什么 → 用改过的本子直接入局对抗 agent。两面闭环。
- **信念 schema 地基**（`from_act`+provenance）→ 认知卡随幕生长 + P4c 完整版。
- **测谎 / 多角色对质**（深化入局，复用 grounding）。
- 生成新故事线/复购（事件溯源 + 某幕分支）；多 agent what-if 重仿真。

> **统一性**：这套把「创作体检台 = 求解器层」「信念 schema = 节点 provenance」「认知矩阵 = 画布一个镜头」「对抗 agent = what-if 重仿真层」收敛进同一张图。

## 6. 理论依据与出处（按需查阅）

> 扎根于叙事 AI / 信息可视化 / 知识表示的**成熟理论**（部分为验证数十年的奠基经典），并对齐 **2024–2025 最新前沿**。以下「前沿在前、奠基在后」。

- **近两年前沿**：Huang et al. ICLR 2024（LLM 自我纠错越改越差）· FlawedFictions 2025（arXiv 2504.11900，LLM 查漏随长度退化、注入 >50% 新漏洞）· PoLL 2024（异构评审团）· Sotopia-ToM（IFC 防泄漏 9.9%→2.2%）
- **求解器/可解性**：clingo / Potassco（ASP）· ASP-for-PCG（Smith & Mateas）
- **表示模型**：Trabasso（因果网络）· Swartjes & Theune（fabula 6 类）· Elson（Story Intention Graph）· Ryan / Kreminski Felt（story sifting）
- **依赖传播/重规划**：Doyle、de Kleer（TMS / ATMS）· AGM（信念修正）· Riedl & Young（IPOCL）· Fox（plan-stability）
- **可视化**：Ghoniem & Fekete（矩阵 vs node-link）· Sugiyama / StoryFlow（分层布局）· Collins & Loftus（激活扩散）
- **库**：@xyflow/react · elkjs · clingo-wasm · Zustand
