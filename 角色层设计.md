# 角色层设计（改本→角色认知 + 加人物）

> 把事件层那套**分工铁律**（LLM 提议 → 确定性引擎划范围 → 求解器盖章 → 仅会话不落盘）扩到**角色/认知层**。A1（影响面）+ B1（加人物）已建成，其余待做。

## 0. 核心判断

我们已有的 **IFC 隔离墙 `G(幕,上帝)\G(幕,角色)` + per-character KB 投影 + 分工铁律**，与学界验证过的最佳实践**高度同构**——不是从零造，有成熟模板可直接抄。

## 1. 四块可直接抄的模板（出处）

- **角色 KB schema** = HeadSpace 三分区 `⟨信为真 B+ / 信为假 B- / 未知 U⟩`，与上帝 World State 分开持有；原生支持假信念（literal 在 B+、世界里实为假）。〔Sanghrajka & Young, HeadSpace, AIIDE 2022〕
- **何幕获知 / 谁告诉谁** = Talk of the Town 信念 facet：Value(可假)/Predecessor/Parents(谁告诉谁)/Evidence/**from_act**/Accuracy(可随世界翻转)。几乎逐字就是我们要的"信念 schema + provenance"。〔Ryan, UCSC PhD 2018〕
- **改本→认知更新** = DEL product update + Bolander「见即信」：谁见证是 STATE 属性(在场/witness)，"j 看见 i 则 j 信 i 带来的 φ"；失败→AGM 式回退到 U。〔Bolander 2018；Eger & Martens AIIDE-17；HeadSpace 失败策略〕
- **加人物（关系/动机一致）** = CiF/Ensemble（社交交换 + 影响规则 + 微理论，规则化可校验、加权 volition）+ Generative Agents（记忆流→反思→检索，消融证明三组件缺一不可）。〔McCoy CiF AIIDE-11；Ensemble；Park et al. UIST 2023〕
- **一致性盖章** = CPOCL-style 意图/冲突/信念叙事规划器，纯 ASP ~200 行可实现；约束如"被意图 fluent 须在某点满足""事件须属其 actor 意图框架"可直接当角色层约束；"部分谓词作输入、求解器补全其余" = 编辑情节→重导出自洽角色状态。〔Dabral & Martens, AIIDE-20〕

## 2. 现状

**已建成**

- **A1 改本→认知·影响面**：改/删/插事件 → 下游因果 BFS + 知识图谱（KNOWS/误信/守秘边）派生哪些角色认知被牵动（在场/知道/误信/守秘 + 具体事实），分直接/下游两档，画布悬停刷亮。纯确定性。`backend/cognition.ts`。
- **B1 加人物·命名充实**：起名(+方向) → LLM 充实成主角量级（5–7 关系 + 4–6 个带类型、跨多幕、因果锚定的剧情线）→ 引擎框定（连真人/落合法幕/真锚点/禁重名）→ 落进关系网 + 故事图。仅会话。`backend/character.ts`、`CharacterForge`、`sessionGraph.ts`。

**接下来**（按依赖序）

- **信念 schema 地基**：B+/B-/U + from_act + provenance（ToTT facet 为模板）——A2 与「加人物可玩」的共同地基。
- **A2 改本→认知·完整可玩**：重放事件 product-update 重算每角色 KB → 用改过的本子直接入局，agent 认知当场变。
- **加人物·可玩**：把 B1 草稿转成迷雾 KB → 新角色能进审问/同台。
- **B2 加人物·全新推荐**：AI 分析阴谋网缺口提议候选（不靠命名）。
- **角色层求解器闭环**：新角色/新剧情接进改本连锁 + ASP 盖章（可知性/时序/关系无矛盾/IFC）；目前只在前端会话层渲染。

## 3. 待confirm

1. **嵌套 ToM**（"凶手以为侦探不知道 X"）——测谎/防泄漏其实已在用。扁平三分区够 v1；真正的"信念之信念"要 possible-worlds DEL，算力会很大，留后面。
2. **NL→谓词抽取**（LLM 中文动机/秘密 → 求解器谓词）是最新、最脆的中间层，需人校对 + 谨慎设计——加人物最大风险点。
3. **求解器层 IFC**（强制"上帝真相不可反推"）无现成系统做过，需自定义 ASP 约束并验证完备性（prompt 层已有 stripReferee 做base）。
