# 角色层设计（改本→角色认知 + 加人物）

> 接续"事件层"（P4 改本/连锁/求解器/应用层）的下一程：把同一套**分工铁律**（LLM 提议 → 确定性引擎划范围 → ASP 求解器盖章 → 仅会话不落盘）扩到**角色/认知层**。
> 本文方案由一轮 deep research 支撑（25 条声明 3-0 核验、0 推翻），下方"模板"列均有一手文献出处。

## 0. 核心判断
我们已有的 **IFC 隔离墙 `G(幕,上帝)\G(幕,角色)` + per-character KB 投影 + 分工铁律**，与学界验证过的最佳实践**高度同构**——不是从零造，有成熟模板可直接抄。

## 1. 四块可直接抄的模板（出处）
- **角色 KB schema** = HeadSpace 三分区 `⟨信为真 B+ / 信为假 B- / 未知 U⟩`，与上帝 World State 分开持有；原生支持假信念（literal 在 B+、世界里实为假）。〔Sanghrajka & Young, HeadSpace, AIIDE 2022〕
- **何幕获知 / 谁告诉谁** = Talk of the Town 信念 facet：Value(可假)/Predecessor/Parents(谁告诉谁)/Evidence/**from_act**/Accuracy(可随世界翻转)。几乎逐字就是我们要的"信念 schema + provenance"。〔Ryan, UCSC PhD 2018〕
- **改本→认知更新** = DEL product update + Bolander「见即信」：谁见证是 STATE 属性(在场/witness)，"j 看见 i 则 j 信 i 带来的 φ"；失败→AGM 式回退到 U。〔Bolander 2018；Eger & Martens AIIDE-17；HeadSpace 失败策略〕
- **加人物（关系/动机一致）** = CiF/Ensemble（社交交换 + 影响规则 + 微理论，规则化可校验、加权 volition）+ Generative Agents（记忆流→反思→检索，消融证明三组件缺一不可）。〔McCoy CiF AIIDE-11；Ensemble；Park et al. UIST 2023〕
- **一致性盖章** = CPOCL-style 意图/冲突/信念叙事规划器，纯 ASP ~200 行可实现；约束如"被意图 fluent 须在某点满足""事件须属其 actor 意图框架"可直接当角色层约束；"部分谓词作输入、求解器补全其余" = 编辑情节→重导出自洽角色状态。〔Dabral & Martens, AIIDE-20〕

## 2. 分阶段路线

### Phase 1 · 轻量、demo 先行（**已实现**）
- **A1 改本→认知·影响面** —— 已建成。改/删/插事件 → 沿下游因果 BFS + 知识图谱（KNOWS/误信/守秘边）派生**哪些角色的认知受牵动**（在场/知道/误信/守秘 + 具体哪条事实），分「直接牵动 / 下游可能波及」两档，画布悬停刷亮。纯确定性、零求解器。落地：`backend/cognition.ts` + `/cognition` + `CognitionDrawer`。
- **B1 加人物·命名充实** —— 已建成。起名(+方向) → LLM 在阴谋网里充实成**主角量级**的人（5–7 条关系 + 4–6 个带类型、跨多幕、因果锚定到现有事件的剧情线）→ 引擎框定（关系连在册真人、事件落合法幕、锚点是真事件、**硬禁用重名** + 换一版 avoid + 截断救援解析）→ 采纳后落进**关系网**（节点+关系线枢纽）**+ 故事图**（带类型事件接进现有因果，矩阵左列新角色泳道）。仅会话。落地：`backend/character.ts` + `/character/suggest` + `CharacterForge` + `sessionGraph.ts`。
  - 与原计划差异：ASP 盖章这一步**尚未接**（新角色/新剧情目前只在前端会话层渲染、确定性框定把关），挪到 Phase 2 的「角色层求解器闭环」。

### Phase 2 · 深做（信念地基 → 可玩 + 全新推荐）
- **地基：信念 schema**（B+/B-/U + from_act + provenance）——一直推迟的"信念地基"，是两个完整版的共同地基（ToTT facet 是模板）。
- **A2 改本→认知·完整可玩**：重放事件 product-update 重算每角色 KB → agent 认知当场变 → 用改过的本子直接入局。
- **B2 加人物·全新推荐**：AI 分析阴谋网缺口 → 提议契合的全新候选角色 → 整捆生成 → 求解器把关。
- 角色层求解器约束：可知性(只知见证/可推的)/时序(from_act)/关系无矛盾/**IFC**(上帝真相不可从角色 KB 反推)。

## 3. 三个诚实的坑（研究点出的）
1. **嵌套 ToM**（"凶手以为侦探不知道 X"）——测谎/防泄漏其实已在用。扁平三分区够 v1；真正的"信念之信念"要 possible-worlds DEL，算力会爆，留后面。
2. **NL→谓词抽取**（LLM 中文动机/秘密 → 求解器谓词）是最新、最脆的中间层，需人校对 + 谨慎设计——加人物最大风险点。
3. **求解器层 IFC**（强制"上帝真相不可反推"）无现成系统做过，需自定义 ASP 约束并验证完备性（prompt 层已有 stripReferee 打底）。
