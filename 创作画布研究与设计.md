# 上帝·创作模式 → 「创作画布」研究与设计（研究支撑版）

> 来源：4 个并行研究 agent（叙事形式化表示 / 创作工具 UX 范式 / 编辑影响传播 / 时序因果可视化），均带一手学术与业界出处。
> 一句话：把"上帝/创作"从**看的工具**升级成**改的工具**——一张**以角色为主线、沿幕推进的可编辑因果故事图**，作者改一处，引擎**确定性推演涟漪范围**（神经网络式逐层激活动画），落到**改本/丰富剧情/查逻辑/对抗推演**。研究一致支持这个方向，且它顺势**统一**了我们之前分散的 roadmap（创作体检台 / 信念 schema / 对抗 agent）。

> **✅ 实现进度（2026-06）**：**P0**事件因果层(68事件/92因果边，`08_story/`+`backend/story.ts`+`/story`) · **P1**总览画布(`StoryCanvas` 幕×角色泳道+卡片+因果边) · **L2**钻取详图(`StoryDetail` React Flow+elkjs；角色/事件/幕三入口) · **P2**影响推演(事件详图三模式：因果邻域[深度1/2]/⚡推演下游[逐波动画+blast-radius]/🔎溯源上游；**焦点锚定 vs 选中**分离、面包屑导航栈、`StoryMinimap`「你在这里」防迷路) 均已建成测通。**下一步 P3**：§3 求解器层(clingo/ASP)把"过度近似的传播高亮"升级成"求解器确认的矛盾/不可解/孤儿知识报警"，标记画到事件详图上。下面 §1–§5 为研究底稿，部分细节随实现微调（如节点先做 8 类、信念/嵌套层后置）。

## 0. 核心结论（先读这条）

1. **表示**：存**一份客观事件因果图** + 每角色**投影**（不复制事件，只在 `(事件,角色)` 上挂可见性/认知标注）——这是全部文献最大共识，且正是我们 `G(幕,视角)` 从"知识切片"扩展到"故事切片"。
2. **呈现**：**幕=时间列(X) × 角色泳道(Y) + 因果有向边**，节点做成**卡片**。**这里用 node-link 图是对的**——因果链/路径追踪恰是 node-link 唯一稳赢的任务（与我们之前"矩阵>节点图"不矛盾：那是稠密属性数据，这是因果路径数据）。矩阵不死，降为其中一个"枢纽总览"镜头。
3. **推演**：改一处→影响范围 = **三层**（确定性依赖传播"框范围" + ASP 求解器"判生死" + 可选 agent"跑 what-if"）。**绝不让 LLM 自评影响**（Huang ICLR 2024）。
4. **统一**：这套**把"创作体检台(可解性/矛盾检测)"、"信念 schema(from_act+provenance)"、"对抗 agent" 全收编进同一个画布**——求解器既做编辑重验、也做逻辑体检；投影需要的 provenance 正是信念 schema；可选重仿真就是对抗 agent。

## 1. 表示模型：一份客观 fabula + per-character 投影

文献三层共识（Trabasso 因果网络 / Swartjes&Theune fabula 模型 / Elson Story Intention Graph）：**(1) 一份客观事件因果网；(2) per-character 投影 = 焦点化(focalization)的 sjuzhet；(3) 事件之上叠 目标/信念/情感 解释层。**

### 节点 taxonomy（建议 10 类）
`Event(客观发生) · Reveal(线索揭示，复用 REVEALS) · Decision(角色决定=Attempt) · Lie(说谎/注入假信念，复用 BELIEVES_FALSELY+MUST_HIDE) · Perception(角色感知到某事) · Goal(动机) · Outcome(角色相信的结果) · RelationChange(关系变化，复用 REL) · Belief(信念状态) · Beat(可选叙事节拍)`。
底座 = Swartjes&Theune 6 类(Goal/Action/Outcome/Event/Perception/Internal-Element) + Elson 的 Goal/Belief 解释层。关键洞见：**Outcome 是"心理的"**（角色*相信*目标达成与否，未必合真相）、**Perception 显式建模"谁看到/听到了什么"**——天然契合剧本杀的假信念/迷雾。

### 边类型
**4 类因果（按强度，Swartjes&Theune）**：φ 物理因果 / m 动机 / ψ 心理因果 / e 使能。
**叙事专用边**：`next`(时序) · `reveals`(线索→事实) · `depends_on/precondition_for`(必要) · `would_cause/would_prevent`(充分/假设，Elson) · `contradicts` · `conflicts_with`(用 CPOCL 7 维标签:参与者/主题/时长/直接性/强度/平衡/解决) · `rel_change`。
**双时间轴**：story-time(客观时序) vs telling-time(揭示顺序)——支持闪回式揭示(Genette order；Elson SIG 双轴交叉引用)。

### per-character 主线投影 = story sifting 查询
理论 = focalization(Bal/Genette)。实操 = **story sifting**(Ryan 2018；Kreminski **Felt** 用 Datalog/DataScript 查询)：把"某角色第2–3幕主线"写成 `actor==X OR target==X OR perceiver==X`，按 `act + t_story` 排序。多角色 = **swimlane 并置**，共享事件跨道连线。**支线** = 高连接度的 **Plot-Unit 子图**(Lehnert)/ Loose-Ends plot thread(Kreminski 2022)。

### 节点字段 schema（草案）
```
Node: id, type, title, summary,
  who: actors[]{role: agent/patient/witness}, when:{act, t_story}, when_told:t_telling, where,
  preconditions[], effects[], facts[], clues[],
  objective:bool, visibility:{char: HIDDEN|PERCEIVED|KNOWN|BELIEVED_FALSE},   // 复用 KNOWS/BF/MUST_HIDE
  status_per_char:{char: HYPOTHETICAL|ACTUALIZED|CEASED},                     // Elson：线索/不在场证明随幕翻转
  belief_owner, belief_subject,                                              // 嵌套信念 X以为Y以为
  affect:{valence,motiv_type}, confidence,                                   // Talk of the Town: strength 随时间衰减
  provenance:{source_char, via: observe|tell|lie|infer, time},               // ← 这就是"信念 schema"
  on_causal_chain:bool, connectivity:int                                     // Trabasso：自动高亮关键线索
Edge: type∈{next, causes_physical(φ),causes_psych(ψ),motivates(m),enables(e),
  reveals, depends_on, would_cause/would_prevent, contradicts, conflicts_with(7维), rel_change},
  from, to, by_character?   // 该边是否仅在某角色主观视角成立
```
**自动辅助作者**：`connectivity`+`on_causal_chain` 自动标"关键线索"；`status_per_char`(Hypothetical→Actualized→Ceased) 让不在场证明随幕翻转；CPOCL `non-executed step` 建模"想做但被阻止"的隐藏意图。

## 2. 创作画布 UX：卡片节点 + 幕列×角色泳道 + 插入即编辑

### 节点 = 卡片（不是圆点），但语义缩放降级
几乎所有成熟工具（articy:draft、Yarn Spinner、Arcweave、Aeon）都把节点做成**富卡片**，让大图不打开即可读。**卡面只放 6 项（头号坑是过载）**：① 类型图标+色（事件/线索/决定/反转）② 标题 ③ 一行 synopsis ④ 涉及角色头像点 ⑤ 状态徽章（草稿/已校验/有冲突）⑥ 依赖/线索计数。完整正文/条件/台词收进**"卡背"详情抽屉**（Trello 两层范式）。**务必语义缩放**：缩小时卡片退化为"色块+标题"（Miro）——这是 Twine"无法缩小看全图而卡死"的正解。给紧凑/舒适切换。

### 多角色时间轴 = Aeon「地铁图」模型
**无人把一张卡片物理拉跨多条泳道**。共享事件两态：**分离视图**中在每条相关泳道各放一张同 ID 卡副本(Aeon Separated)；**全局/地铁视图**中收敛成**多轨交汇的一个交叉站**(Aeon Subway)。布局学 **StoryFlow**(Liu et al. TVCG 2013)：幕=离散时间层，层内对角色线**排序+对齐最小化交叉**、连续段**压缩最小化 wiggle**。**默认聚焦单角色**(Causality 式按角色过滤)，跨角色因果连线**只在聚焦/悬停时显示**。

### 插入即编辑 + 重构预览（关键差异化）
主手势学 Houdini/Blender/React Flow：**连线中点常驻"+"**(React Flow button-edge) → 拖新卡到线上 → **自动断旧线、上接前置、下接后继** → **自动布局让后续节点顺移让位**(Figma 兄弟让位)。**杀手锏 = 借 IDE 重构预览**：落定前用**幽灵态 + diff 高亮**告诉作者"这会把 X 的后续 3 个节点右移、与线索 Z 产生新依赖、可能影响结局"，逐项确认再提交（呼应 Causality"违规即报警"）。**把"丰富情节"从制造混乱变成可控操作**——这正好接第 3 节的影响推演。

### 防 hairball 五条硬规则
(1) **分层下钻**：幕/场作容器，顶层看幕级摘要，双击进入看节点级（绝不像 Twine 全摊单板）；(2) **聚焦优先**：默认单角色单幕，连线按需显隐；(3) **语义缩放**降级卡片；(4) **逻辑显形**：把"浑浊变量"做成卡面可见的依赖/冲突徽章；(5) **branch-and-bottleneck**：鼓励分支定期汇合，从结构上压制组合爆炸(Emily Short / Ashwell)。
**失败教训**：Twine 700+ 节点卡死 + 逻辑隐形；inkle 的反节点论（"想在分支中间加一个选项得把整张图拆开挪"——正是我们要用"插入即编辑+自动布局"解决的）；articy 学习曲线陡。

## 3. 改一处 → 算影响范围（三层，不是三选一）

**确定性依赖图"框范围"（快/全/可解释） + 外部求解器"判生死"（权威） + agent"跑 what-if"（可选，只在社交涌现层且回写复核）。绝不让 LLM 自评影响。**

### 层一·确定性依赖传播（这是"神经激活"高亮的骨架）
- **TMS/ATMS**(Doyle 1979 / de Kleer 1986)：每个事实/信念节点记录 **justification**(支撑它的前提集)；改/删一个前提 → 沿 justification 图传播失效。**ATMS label = 最小假设集**，天然匹配多视角；**nogood = 矛盾位置**=可解释根因。**把上帝视角真相图做成 ATMS 的 assumption 基。**
- **增量重算**：推理链写成 Datalog/ASP 规则(`reveals(C,F),holds(C)=>holds(F)`)，改动作为 delta 注入，只重算受影响子图（Differential Dataflow / DRed 半朴素）。
- **信念修正 = AGM 最小改动**，用 **belief base**(有限显式基，可追责) 而非演绎闭包。**迷雾即作用域剪枝**：只对"改动因果可达 + 本应感知到"的角色触发 revision；**改动在某角色迷雾之外 → 其信念不变**（天然剪枝）。`BELIEVES_FALSELY` 是**故意的不一致**，登记为豁免 nogood，不可当 bug 报。

### 层二·外部求解器判定（= 创作体检台）
把"剧本可解"编码成 **ASP(clingo/Potassco)** 或 SAT/CSP：硬约束 = "每个谜底有支撑链可达"、"凶手唯一可推"、"时序无悖论"。改动后重跑：**有解=仍可解；UNSAT core=精确定位被破坏的约束**。ASP 的非单调语义尤其匹配"侦探推理+缺省假设"。**一个 clingo 引擎同时覆盖：一致性检查 + 可解性 + UNSAT 定位**——性价比最高的外部锚。
- **FlawedFictions**(arXiv 2504.11900) 实证：LLM 在 plot-hole 检测随故事变长**显著退化**、且生成本身注入 >50% 更多漏洞 → **不能让 LLM 当一致性裁判**；但其受控注入思路可做**回归测试集**。

### 层三·叙事重规划（断链时最小代价修圆）
**IPOCL**(Riedl&Young 2010) 因果链 = 我们的依赖边；**Plan Repair > Replan**(Fox 2006，plan-stability 度量 = 涟漪大小的现成指标)：先局部修补受影响因果链，失败再 mediation 式 replan(Robertson&Young：accommodation = 插入新事件后自动把后面修圆)。

### Blast Radius 度量（高亮什么、报警什么）
从改动节点 BFS 求 **impact set**。量化六维：① 受影响节点数 ② 波及角色数(几个 belief base 被 revise) ③ **幕跨度** ④ 关系边变更数 ⑤ **是否触及结局/谜底**(∩终局支撑链≠∅→标红高危) ⑥ plan-stability 距离。**over-approx(图可达=可能影响，用于高亮) vs 真实(求解器确认改变可解性，用于报警)。**

### 最小管线
建依赖图(ATMS+真相图为 assumption 基，推理链=Datalog/ASP) → 收编辑 delta → 确定性 BFS 传播(impact set，按迷雾过滤) → belief base revision → **clingo 重验**(SAT=可解 / UNSAT core=矛盾位置) → plan repair(若断链) → 度量+高亮(**绿=未触及 / 黄=认知变更 / 红=破坏可解性或触及谜底**)。**LLM 仅为红/黄项生成人话解释，不参与判定。**

## 4. 可视化 + 「神经激活」传播动画

### 布局与库
**幕=时间列(X) + 角色泳道(Y) + 因果有向边**，**StoryFlow 式**"先定时间层、再层内排序压缩"最小化交叉与抖动(根：Sugiyama 1981 分层绘图)。**PlotThread**(VIS 2020) 的"作者+AI 共享画布混合主动编排"正是我们"作者改、引擎推演"的范式。
**库 = React Flow/@xyflow**（节点=任意 React 卡片组件 + MIT + 官方 dagre/ELK）；主布局用 **ELK `layered`**（端口+约束，质量优于 dagre）；上万节点再降级 **AntV G6**(Canvas/WebGL)。

### node-link 在这里是对的（纠偏的纠偏）
Ghoniem/Fekete 2004 原文：矩阵在 >20 节点多数任务胜，**唯 path finding 始终偏向 node-link**；NodeTrix/Okoe 复现。我们的"因果链/影响传播"恰是 path/connectivity 任务 → **node-link/分层图就是对的**。仅当图**极稠密**才退化。**仍配矩阵/时间线联动**：node-link 追因果、矩阵看"谁是关系枢纽"、时间线定位幕（NodeTrix 式内嵌 / brushing-and-linking）。

### 「神经激活」传播动画（demo 爆点）
理论 = **spreading activation**(Collins&Loftus 1975)。配方：
- **(a) BFS 分层**：从被改节点跑 BFS，每个 frontier(等跳距集)=一道波；与时间层天然对齐（第 k 跳在第 k 列点亮）。
- **(b) 沿边动画**：React Flow `<AnimatedSvgEdge>` 粒子流 / SVG `animateMotion` + `stroke-dashoffset` "通电流动"；编排用 **GSAP Timeline.stagger** 逐层延迟，或 D3 `.delay(d=>d.bfsLevel*step)`。海量边走 deck.gl TripsLayer。
- **(c) 传达"影响多大"**：① 透明度随跳距衰减(`opacity=D^k`)；② **扩张光环 scale ∝ 可达节点数**；③ 角标 **blast-radius 数字徽章 = |可达集|**；④ 边粗 ∝ 传播激活(TensorFlow Playground 风格)。
### 防糊（按收益排序）
按 facet 过滤(幕/角色，**Overview first, zoom & filter**) → Ego/DOI 聚焦(Furnas/van Ham&Perer) → 折叠聚合 + 语义缩放(Motif Simplification) → 边捆绑(仅导出总览，慎用)。

## 5. 这如何统一我们的 roadmap（关键）

| 之前分散的事 | 在"创作画布"里的归宿 |
|---|---|
| **创作体检台**(可解性/矛盾检测) | = 层二 **ASP 求解器**——既做编辑重验、也做静态逻辑体检 |
| **信念 schema**(from_act + evidence) | = 节点 schema 的 `provenance` + `status_per_char`；上一次缓存的"认知卡随幕生长"也靠它 |
| **认知矩阵** | 降为画布里的一个**联动镜头**("谁是关系/知识枢纽"总览)，不再是上帝模式的全部 |
| **对抗多 agent** | = 层三**可选 what-if 重仿真**（改动后跑一段社交涌现，回写并由求解器复核） |
| **时间轴 ActScrubber** | 升级为画布的 X 轴本体（幕=时间列） |

→ 也就是说：**这一个方向把上帝/创作模式 + 第二阶段地基 + 第三阶段对抗，收敛成同一张画布的不同层。**

## 6. 分阶段落地建议（demo 优先 → 严谨为后）

- **P1·故事图投影 + 卡片画布（最快出效果）**：从现有数据投影"事件/揭示×角色×幕"的故事图，幕列+角色泳道+卡片节点(React Flow + ELK)，单角色聚焦 + 语义缩放。**先能看、能读、能聚焦。**
- **P2·确定性依赖传播 + 激活动画（demo 爆点，且可靠）**：建 justification 依赖图，改/选一个节点 → BFS 分层 → 神经激活动画 + blast-radius 徽章。**这步不需要 LLM、不需要求解器就很惊艳，且诚实（确定性）。**
- **P3·ASP 求解器重验（= 创作体检台）**：clingo 编码可解性/矛盾/时序，编辑后重跑，UNSAT core 定位 → 红黄绿高亮。**严谨的逻辑护栏在这步立住。**
- **P4·插入即编辑 + 重构预览**：连线"+"插卡、自动布局、提交前 diff 影响预览。
- **P5·可选 what-if 多 agent 重仿真**：改动后跑社交涌现，回写复核。

## 7. 关键数据缺口（要先补的）
现有图谱有 fact/clue，但**缺一个显式的"事件因果层"**（带 preconditions/effects/causal 边、per-character 参与与可见性）。需从 `00_index/timeline.md`(事件×幕×涉及人物×事实ID)、`05_acts/*`(幕流程)、`02_truth/*`(真相) **抽取/编纂出 Event 节点与因果边**。这是 P1 的前置，也是"信念 schema/provenance"落地的同一件事。建议先做一个小型 Event 抽取，跑通"投影→画布→传播"的纵切。
