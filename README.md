# 流氓叙事 · 叙事引擎（Demo）

把一个真实剧本杀（《流氓叙事》，2281 页纯图本）数字化成 **时序、多视角的知识图谱**，并在其上演示「**一个引擎，两种投影**」——核心抽象 `G(幕, 视角) → 子图`。

护城河不是 AI，而是这套**结构化、可计算**的模型：引擎在每一幕都算得清「每个角色知道什么、被瞒什么、手上有哪些线索、要守哪些秘密」。

> 设计文档见 `引擎与Demo设计.md`；可视化/数据模型研究见 `可视化研究与方案.md`；创作画布研究与设计见 `创作画布研究与设计.md`。

---

## 两种模式（一引擎两面）

界面由顶部「**现场 ⇄ 上帝**」切换，底部一根「**幕**」时间轴统驭一切（滑到哪一幕，全部视图更新到哪一幕）。

### 🎭 现场 · 入局（玩家视角 · 迷雾）
- **关系网当主舞台**：点任一角色 = 「成为 TA」，图进入迷雾、聚焦 TA 的关系。
- 右栏 **认知卡**：TA 这一幕——
  - **线索**：DM 各幕发给 TA 的实体道具卡，标「本幕新发」，展开可看**真实卡面内容**（OCR）。
  - **认知**：信念（真/误信）、必须守住的秘密；创作者可「窥视墙后」看假信念背后的真相。
  - **本幕**：目标 / 台词 / 沉浸正文。
  - **审问**：直接对话——agent 只拥有 TA 的主观认知，**结构上吐不出墙后真相**（防泄漏审计可证）。
- → 这是**对抗 agent** 的 demo 面。

### 🪄 上帝 · 创作（创作者视角 · 全知）= 创作画布
把本子拆成 **68 个事件 + 92 条因果边** 的故事图，作者在其上**读懂 / 编辑 / 推演**。

- **L1 总览画布**：`幕(列) × 角色(泳道)` 的卡片故事图——每张卡=一个事件（颜色=类型：决定/谎言/揭露…），连线=因果（导致/动机/使能/矛盾）。点角色泳道→钻进 TA 的故事线，点幕头→该幕因果切面。
- **L2 钻取详图**（ELK 分层 · 富卡片）：**一个事件 = 一张图**——前因(左) ← **焦点**(中) → 后果(右)。
  - **因果邻域**（深度 1/2 跳可调）：读懂它的来龙去脉。
  - **⚡ 推演下游**：若改动此事件，沿因果链**逐波点亮**下游（神经网络式涟漪）+ **影响范围**（波及 N 事件 / 牵动 M 角色 / 幕跨度 / ⚠ 触及结局），纯确定性、零 LLM。
  - **🔎 溯源上游**：追到它的根因（从根因向焦点逐层推进）。
- **防迷路**：面包屑栈（可跳回任意一步）+ 「你在这里」故事全景图；**锚点(本图焦点) vs 选中(看信息)** 分离——单击别的节点只看信息、锚点不变，要换焦点才开新图。
- → 这是**改本 / 丰富剧情 / 查逻辑 / 看改动涟漪** 的 demo 面（求解器精确报警为下一步）。

## 架构（前后端分离 · pnpm monorepo）

```
frontend/   Vite + React 19 + React Flow + elkjs + 自绘 SVG + Tailwind + Zustand   纯 UI
backend/    Node + Hono                                                          图编译/查询 + 对话(DeepSeek) + 线索 + 故事图
shared/     Zod schema + sliceGraph(G(幕,视角)) + StoryGraph                      前后端单一事实源
digitized/  数据（流氓叙事，已全本数字化）+ _ocr（OCR 文本）+ 08_story（事件因果层）
```

- **知识图谱** `backend/src/compile.ts`：读 `digitized/流氓叙事/` → **324 节点 / 205 事实 / 85 线索 / 125 KNOWS / 14 假信念 / 11 守秘 / 99 揭示 / 62 关系** 边。
- **故事图（创作画布）** `backend/src/story.ts`：读 `08_story/{events,causal_edges}.yaml` → **68 事件 + 92 因果边**（从裁判文档 `阴谋网.md`/`timeline.md` 提取）。
- **防泄漏** `backend/src/chat.ts`：`knowledge.yaml` → system prompt，剥离裁判真相 + leak 审计；**命门建议问题** + **每句回答挂依据/墙后**。
- **线索内容** `backend/src/clues.ts`：`clues.csv` 元数据接上 `_ocr/线索/*.pages.json` 的真实卡面文字。
- **认知卡** `backend/src/dossier.ts`：角色人设/信念/秘密/关系/目标/线索/本幕正文。
- **LLM**：DeepSeek（`deepseek-chat`=flash，可在 `.env` 切 `deepseek-reasoner`=pro）。

## 运行

前置：Node 22 + pnpm（已装）。`.env` 已含 DeepSeek key（不入库）。

```bash
pnpm install            # 安装依赖
pnpm build:shared       # 构建 shared（前后端共用类型）

pnpm dev                # 一条命令起全栈（backend:8787 + frontend:5173）
# 或分别：pnpm dev:backend / pnpm dev:frontend
```

打开 **http://localhost:5173**。

## Demo 走法（5 分钟）

1. **现场模式（默认）** — 主舞台是人物关系网，底部幕轴停在第三幕。点关系网里的人、或左上角色 pill，**进入 TA 的视角**。
2. **看认知卡** — 进入「程走柳」，右栏「线索」tab：DM 发给她的卡按幕列出，展开第三幕那张 **Yanni 唱片** → 真实歌词 + 暗藏手写台词「走柳，等我离开房间，出来找我一下。」
3. **审问** — 切「审问」tab，问「阿奇在哪」「有人说你杀了阿奇」——角色在戏里回避，**结构上无法承认**；顶部审计「leaked:false」是证据。
4. **拖底部幕轴** — 滑到不同幕，关系网迷雾 / 认知卡 / 线索随剧情变化；每幕的「发牌密度」（蓝事实/绿线索）一目了然。
5. **切「上帝 · 创作」= 创作画布** — 幕×角色泳道的故事图。**双击**任一事件（如「建电台」）→ 进它的因果详图（前因左/焦点中/后果右）。
6. **看改动涟漪** — 在详图里点「⚡ 推演下游」→ 沿因果链逐波点亮，HUD 显示「波及 17 事件 · 牵动 17 角色 · 幕 5→12 · ⚠ 触及结局」；点「🔎 溯源上游」→ 追到根因。右下「你在这里」+ 面包屑随时定位、不迷路。

## 接口

- `GET /graph` — 编译好的全图
- `GET /character/:name?act=` — 角色认知卡（人设/信念/秘密/关系/目标/**线索含内容**/正文）
- `GET /clue/:id` — 单条线索 + OCR 真实卡面内容
- `GET /act/:ord` — 某幕 DM 主持流程
- `GET /story` — 创作画布故事图（68 事件 + 92 因果边 + cast）
- `GET /probe/:name?act=` — 命门审问问题（按角色×幕生成）
- `POST /chat` `{character, actName, messages}` — 角色对话 + leak 审计 + grounding（依据/墙后）
- `GET /kb/:character?act=` — 查看某角色编译出的 system prompt + 审计（调试/举证）

## 进度 & roadmap

创作画布主线（详见 `创作画布研究与设计.md`）：
- ✅ **P0** 事件因果层（68 事件 / 92 因果边）· ✅ **P1** 总览画布 · ✅ **L2** 钻取详图（角色/事件/幕 + 焦点锚定/三模式/导航栈/你在这里地图）· ✅ **P2** 影响推演动画（推演下游 + 溯源上游，纯确定性）
- ✅ **P3 创作体检台**：图层（矛盾/因果环/缺前因，`audit.ts`）+ **clingo ASP 求解器**（时序一致性/可解性可达，`solver.ts`）双层、皆外部锚定不靠 LLM。`/audit` + 「逻辑体检」面板，发现可点击定位到事件图。
- ⏭ **P4** 插入即编辑（连线插卡 + 提交前 diff 影响预览 + 求解器重验）· **P5** 可选多 agent what-if 重仿真。

其它路线：
- **生成新故事线 / 复购**：事件溯源 + 某幕分支，世界与人设不变、重算下游。＝解决「换大纲」成本。
- **对抗多 agent**：多角色 AI 互相博弈 / 被审问，导演-裁判循环 + 防泄漏靠上下文隔离（现场模式已是底座）。
- **数据模型地基**（信念 schema `from_act`/IFC/因果轴 → 认知卡随幕生长 + 让 P3 体检精准）详见 `可视化研究与方案.md` §2。

> 铁律：LLM 自检不可靠（Huang ICLR 2024）→ 检查/推演**外部锚定**（真求解器 + 真相图谱当参照 + 异构评审团），不让 LLM 自评。

## 已知取舍

- 角色视角"知识随幕增长"较弱（多数角色知识是 act=null 的背景）；线索/事实揭示的幕推进最明显。
- 内存图 + 前端过滤（零数据库），多本子 / 多人 session 时再上图库。
- 仅 F-id 化的信念/线索进图；纯描述性信念在 KB（喂 agent）但不画进图。
