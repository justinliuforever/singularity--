# 流氓叙事 · 叙事引擎（Demo）

把一个真实剧本杀（《流氓叙事》，2281 页纯图本）数字化成 **时序、多视角的知识图谱**，并在其上演示「**一个引擎，两种投影**」——核心抽象 `G(幕, 视角) → 子图`。

护城河不是 AI，而是这套**结构化、可计算**的模型：引擎在每一幕都算得清「每个角色知道什么、被瞒什么、手上有哪些线索、要守哪些秘密」。

> 设计文档见 `引擎与Demo设计.md`；可视化/数据模型研究见 `可视化研究与方案.md`。

---

## 两种模式（一引擎两面）

界面由顶部「**现场 ⇄ 上帝**」切换，底部一根「**幕**」时间轴统驭一切（滑到哪一幕，全部视图更新到哪一幕）。

### 🎭 现场 · 入局（玩家视角 · 迷雾）
- **关系网当主舞台**：点任一角色 = 「成为 TA」，图进入迷雾、聚焦 TA 的关系。
- 右栏 **认知卡**：TA 这一幕——
  - **线索**：DM 各幕发给 TA 的实体道具卡，标「本幕新发」，展开可看**真实卡面内容**（OCR）。
  - **认知**：信念（真/误信）、必须守住的秘密；创作者可「窥视墙后」看假信念背后的真相。
  - **本幕**：目标 / 台词 / 沉浸正文。
  - **审问**：直接对话——agent 只拥有 TA 的主观认知，**结构上吐不出墙后真相**（防泄漏审计可证）。
- → 这是**对抗 agent** 的 demo 面。

### 🪄 上帝 · 创作（创作者视角 · 全知）
- **认知矩阵当主台**：行=6 个 PC × 列=102 条在场事实，四态一屏看尽——
  - <kbd>蓝✓</kbd> 知道 · <kbd>金🔒</kbd> 知道且须隐瞒 · <kbd>红✗</kbd> 被骗（假信念） · 空=墙后不知；底部「客观真相」行=全场可揭示的真相。
- 「按阵营」让知识同盟自动聚成色块；「只看假信念」收敛到高潮列；拖底部幕轴看矩阵随剧情逐列点亮。
- → 这是**帮作者查逻辑 / 可解性**的 demo 面（漏洞检测为下一步）。

## 架构（前后端分离 · pnpm monorepo）

```
frontend/   Vite + React 19 + React Flow + 自绘 SVG + Tailwind + Zustand   纯 UI
backend/    Node + Hono                                                   图编译/查询 + 角色对话(DeepSeek) + 线索/档案
shared/     Zod schema + sliceGraph(G(幕,视角))                            前后端单一事实源
digitized/  数据（流氓叙事，已全本数字化）  +  _ocr（40 个 PDF 的 OCR 文本）
```

- **图编译** `backend/src/compile.ts`：读 `digitized/流氓叙事/` → **324 节点 / 205 事实 / 85 线索 / 125 KNOWS / 14 假信念 / 11 守秘 / 99 揭示 / 62 关系** 边。
- **防泄漏** `backend/src/chat.ts`：`knowledge.yaml` → system prompt，剥离一切 truth/裁判注释，附 leak 审计（`leaked:false`）。
- **线索内容** `backend/src/clues.ts`：`06_clues/clues.csv`（实体道具卡元数据）接上 `_ocr/线索/*.pages.json` 的真实卡面文字。
- **认知卡** `backend/src/dossier.ts`：角色人设/信念/秘密/关系/目标/线索/本幕正文。
- **LLM**：DeepSeek（`deepseek-chat`=flash，可在 `.env` 切 `deepseek-reasoner`=pro）。

## 运行

前置：Node 22 + pnpm（已装）。`.env` 已含 DeepSeek key（不入库）。

```bash
pnpm install            # 安装依赖
pnpm build:shared       # 构建 shared（前后端共用类型）

pnpm dev                # 一条命令起全栈（backend:8787 + frontend:5173）
# 或分别：pnpm dev:backend / pnpm dev:frontend
```

打开 **http://localhost:5173**。

## Demo 走法（5 分钟）

1. **现场模式（默认）** — 主舞台是人物关系网，底部幕轴停在第三幕。点关系网里的人、或左上角色 pill，**进入 TA 的视角**。
2. **看认知卡** — 进入「程走柳」，右栏「线索」tab：DM 发给她的卡按幕列出，展开第三幕那张 **Yanni 唱片** → 真实歌词 + 暗藏手写台词「走柳，等我离开房间，出来找我一下。」
3. **审问** — 切「审问」tab，问「阿奇在哪」「有人说你杀了阿奇」——角色在戏里回避，**结构上无法承认**；顶部审计「leaked:false」是证据。
4. **拖底部幕轴** — 滑到不同幕，关系网迷雾 / 认知卡 / 线索随剧情变化；每幕的「发牌密度」（蓝事实/绿线索）一目了然。
5. **切「上帝 · 创作」** — 认知矩阵一屏看尽全场信息差。点「只看假信念」→ 谁被骗了什么；点「按阵营」→ 知识同盟聚成色块。

## 接口

- `GET /graph` — 编译好的全图
- `GET /character/:name?act=` — 角色认知卡（人设/信念/秘密/关系/目标/**线索含内容**/正文）
- `GET /clue/:id` — 单条线索 + OCR 真实卡面内容
- `GET /act/:ord` — 某幕 DM 主持流程
- `POST /chat` `{character, actName, messages}` — 角色对话 + leak 审计
- `GET /kb/:character?act=` — 查看某角色编译出的 system prompt + 审计（调试/举证）

## 接下来要做的（roadmap）

- **上帝/创作模式做扎实**：矩阵翻身成「信息设计体检台」——自动标红**逻辑漏洞**（孤儿知识：某角色"知道"却没有任何线索/场景给到 TA；无解推理；矛盾），用真求解器（ASP/SAT）外部锚定。＝帮写本子的人查逻辑的核心价值。
- **生成新故事线 / 复购**：事件溯源 + 在某幕分支，世界与人设不变、重算下游。＝解决「换大纲」成本。
- **对抗多 agent**：多角色 AI 互相博弈 / 被玩家审问，导演-裁判循环 + 防泄漏靠上下文隔离。
- 数据模型地基（信念 schema / 知识分区 IFC / 时间因果轴）详见 `可视化研究与方案.md` §2。

## 已知取舍

- 角色视角"知识随幕增长"较弱（多数角色知识是 act=null 的背景）；线索/事实揭示的幕推进最明显。
- 内存图 + 前端过滤（零数据库），多本子 / 多人 session 时再上图库。
- 仅 F-id 化的信念/线索进图；纯描述性信念在 KB（喂 agent）但不画进图。
